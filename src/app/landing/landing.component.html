This page documents the solution to the three challenges provided. Each problem is solved using custom techniques and integrated into the ContractIQ app, which is designed to interrogate legal contracts using Natural Language Processing (NLP) and provide key information extraction.

1. Legal Contract Interrogation with Retrieval-Augmented Generation (RAG) üß†

The first challenge involved writing a Python program that uses RAG to interrogate legal contracts in `.docx` format. The goal was to answer natural language queries using the contract as the primary source of information.

Key Requirements:

  • Implemented at least two techniques to mitigate hallucinations.
  • Custom chunking and parsing techniques used without leveraging external abstractions such as Langchain.
  • Query expansion/transformation technique implemented to ensure robustness in answering questions.
  • Summary of potential hybrid search paradigm using both dense and sparse techniques for a production use case.

Solution Overview:

The solution leverages the OpenAI API for text completion, with custom chunking of the contract to ensure effective retrieval of information. To mitigate hallucinations, we applied two specific techniques:

  • Selective Chunking: We break down the contract into smaller sections based on logical sections (e.g., clauses, paragraphs) and provide context-aware queries to avoid irrelevant information from being generated.
  • Cross-Verification: Each generated response is verified against multiple contract sections to ensure consistency and reduce hallucination risks.

Query expansion was implemented using synonym matching and phrase rewording, ensuring that a wider range of questions can be effectively answered. In a production use case, we would also explore hybrid search paradigms that combine **dense retrieval** (for semantic search) with **sparse retrieval** (for keyword-based precision) to improve robustness and accuracy.

2. Contract Information Extraction Using LLM üìë

The second challenge was to extract three key pieces of information from a contract in PDF form: the effective date, governing law, and parties involved. This was accomplished using a Language Learning Model (LLM).

Key Requirements:

  • Extracted the effective date, governing law, and the parties involved in the contract.
  • Used function calling/tool use for information extraction without relying on structured outputs like JSON.

Solution Overview:

The solution used an LLM with function-calling capabilities to extract specific information. The program read the contract text, identified key sections based on predefined patterns (e.g., regex for dates), and extracted the relevant information:

  • Effective Date: Detected using date pattern matching and contextual indicators.
  • Governing Law: Extracted from sections labeled "Governing Law" or similar headings.
  • Parties: Identified using common contractual language, including terms like "Party A" or "Party B."

The function-calling method allowed for custom logic to be applied based on the contract‚Äôs text, ensuring accurate extraction without relying on rigid structures.
